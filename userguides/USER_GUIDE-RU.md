# 📕 ST Memory Books — Ваш ИИ-ассистент памяти

**Превратите бесконечные переписки в организованные, доступные для поиска воспоминания!**

Нужно, чтобы бот помнил детали, но контекст чата слишком велик? Хотите автоматически отслеживать важные сюжетные повороты, не делая заметки вручную? **ST Memory Books** делает именно это — расширение наблюдает за вашими чатами и создаёт умные сводки, чтобы вы не теряли нить истории.

(Нужны технические детали? Смотрите [Как работает STMB](howSTMBworks-ru.md).)

## 🚀 Быстрый старт (Первое воспоминание за 5 минут)

1. Найдите расширение Memory Books в меню расширений.
2. Включите **Автоматически создавать сводки памяти**.
3. Поставьте **Интервал Авто-Сводки** примерно на `20-30`.
4. Для начала оставьте **Буфер Авто-Сводки** небольшим, например `0-2`.
5. Сначала создайте одну ручную память, чтобы чат был «primed».

После этого STMB будет автоматически создавать воспоминания по мере разговора.

---

## 💡 Что делает ST Memory Books

### Автоматические сводки

STMB следит за чатом в фоне и создаёт памяти через заданный интервал сообщений.

### Ручное создание памяти

Вы можете отметить важную сцену стрелками (► ◄) и создать память только для неё.

### Side Prompts

Side Prompts работают как трекеры и помогают следить за отношениями, квестами, настроением и состоянием мира.

## 🎯 Выберите свой стиль

### Поставил и забыл

1. Включите `Автоматически создавать сводки памяти`.
2. Подберите `Интервал Авто-Сводки`.
3. При необходимости добавьте небольшой `Буфер Авто-Сводки`.

### Ручной контроль

1. Отметьте начало и конец сцены стрелками.
2. Откройте Memory Books.
3. Нажмите `Create Memory`.

### Slash-команды

* `/creatememory` - создать память из отмеченной сцены
* `/scenememory X-Y` - создать память по диапазону сообщений
* `/nextmemory` - взять сообщения с момента последней памяти
* `/sideprompt "Name" {{macro}}="value" [X-Y]` - запустить Side Prompt
* `/sideprompt-on "Name"` или `/sideprompt-off "Name"` - включить или выключить Side Prompt
* `/stmb-set-highest <N|none>` - сдвинуть базовую точку авто-сводки

## 🙈 Экономия токенов: скрыть / показать

Скрытие сообщений не удаляет их. Оно только убирает их из обычного контекста, который видит ИИ.

### Когда это полезно

* чат стал очень длинным
* старые сообщения уже превращены в память
* вы хотите держать чат визуально чище

### Автоскрытие после памяти

* `Не скрывать автоматически` - ничего не скрывать
* `Автоматически скрывать все сообщения до последней памяти` - скрывать всё, что уже покрыто памятью
* `Автоматически скрывать только сообщения последней памяти` - скрывать только последнюю обработанную область

### Показ перед генерацией

`Показывать скрытые сообщения для генерации памяти (выполняет /unhide X-Y)` временно выполняет `/unhide X-Y` перед созданием памяти.

### Базовая настройка

* `Автоматически скрывать только сообщения последней памяти`
* оставить `2` сообщения видимыми
* включить `Показывать скрытые сообщения для генерации памяти (выполняет /unhide X-Y)`

## 🌈 Сводки консолидации

Консолидация помогает сжимать старые STMB-памяти в более высокоуровневые сводки.

### Что это такое

STMB может объединять существующие памяти и сводки в более компактную запись. Первый уровень - `Arc`, далее идут `Chapter`, `Book`, `Legend`, `Series` и `Epic`.

### Когда это использовать

* список воспоминаний стал слишком длинным
* старым сценам больше не нужна подробность по сообщениям
* нужно экономить токены без потери связности

### Автоматически ли это работает

Нет. Консолидация всегда требует подтверждения.

* можно открыть `Объединить воспоминания` вручную
* STMB может показать подтверждение, когда уровень достигает минимума
* выбор `Yes` открывает только окно консолидации, ничего не запускается втихую

### Что консолидация объединяет

* обычные STMB-памяти
* сводки более высокого уровня
* Side Prompts не входят в Arc/Chapter

### Как использовать

1. Нажмите `Объединить воспоминания`
2. Выберите целевой уровень
3. Выберите исходные записи
4. При необходимости отключите исходники после создания
5. Нажмите `Run`

Если AI вернёт плохой результат, его можно просмотреть и исправить перед повторной отправкой.

## 🎨 Side Prompts и шаблоны

Side Prompts - это фоновые трекеры. Они создают отдельные записи в lorebook и могут работать параллельно с памятью.

### Как это работает

* `Prompt`, `Response Format` и `Title` поддерживают стандартные макросы ST
* нестандартные `{{...}}` макросы становятся обязательными для ручного запуска
* Side Prompts могут возвращать обычный текст, JSON не обязателен
* Side Prompts обновляют одну и ту же запись, а не создают новую цепочку записей

### Важно

Если шаблон содержит пользовательские runtime-макросы, он становится только ручным и не должен использовать автоматические триггеры.

### `/sideprompt`

* `X-Y` необязателен
* если диапазон не указан, STMB использует сообщения с последней точки обновления этого Side Prompt

## 🧠 Regex

STMB может запускать выбранные Regex-скрипты перед отправкой текста в AI и перед сохранением результата.

### Для чего это нужно

* чистить повторяющиеся фразы
* нормализовать имена и термины
* править текст до того, как STMB его разберёт

### Как это работает

1. Создайте скрипты в расширении `Regex`.
2. В STMB включите `Использовать регулярные выражения (расширенный)`.
3. Нажмите `📐 Настроить регулярные выражения…`.
4. Выберите, какие скрипты запускать до отправки и до сохранения.

### Важно

* выбор Regex для STMB управляется внутри STMB
* скрипт может запускаться даже если он выключен в самом Regex
* STMB поддерживает множественный выбор для входящей и исходящей обработки

## ⚙️ Важные настройки

Это не полный список. За полной справкой смотрите [readme.md](readme.md).

* `Автоматически создавать сводки памяти` - включает автоматическое создание памяти
* `Интервал Авто-Сводки` и `Буфер Авто-Сводки` - управляют моментом автосоздания
* `Show memory previews` - позволяет проверять/править ответ AI перед сохранением
* `Спрашивать о консолидации, когда уровень готов` и `Уровни авто-консолидации` - показывают возможность консолидации, не запускают её молча
* `Включить ручной режим Лора` и `Автоматически создавать лорбук, если он не существует` - управляют тем, куда сохраняются воспоминания
* `Использовать регулярные выражения (расширенный)` - открывает управляемый STMB выбор Regex
* `Текущие настройки SillyTavern` - использует текущее подключение ST напрямую

## 🔧 Решение проблем

Это не полный список неполадок. За полной справкой смотрите [readme.md](readme.md).

* Убедитесь, что STMB включён и пункт `Memory Books` виден в меню расширений
* Если авто-сводка не срабатывает, сначала создайте одну ручную память и проверьте interval/buffer
* Если память не сохраняется, убедитесь, что чат привязан к lorebook или включён `Автоматически создавать лорбук, если он не существует`
* Если Regex ведёт себя странно, проверьте выбор в `📐 Настроить регулярные выражения…`
* Если консолидация не предлагается, проверьте `Спрашивать о консолидации, когда уровень готов` и `Уровни авто-консолидации`
